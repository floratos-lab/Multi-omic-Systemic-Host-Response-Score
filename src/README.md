# src/ — Analysis Scripts

This folder contains all R analysis scripts for the MoSS discovery and
validation pipeline. Scripts should be run in the order listed below.

Source data files referenced by these scripts are **not distributed** with this
repository. See [Data Availability and Access](../README.md#data-availability-and-access) in the root README for
instructions on obtaining each dataset. Intermediate result files (in
`results/`) are also not distributed; they are generated by running the scripts
in sequence, as documented in each script's header.

---

## Utilities

### `methods.R`
Shared utility functions sourced by other scripts. Not run directly.

### `convertMOFAdata.R`
Converts MOFAdata gene sets from Ensembl to NCBI (Entrez) IDs. Run once before
running `ssEnrichmentScores.R`. Converted gene set files are written to
`annotations/` and are included in the repository.

---

## Step 1 — Enrichment scoring

### `ssEnrichmentScores.R`
Computes GSVA/ssGSEA single-sample pathway enrichment scores from
VST-normalized RNA-seq data. 

**Source data required (obtain per [Data Availability and Access](../README.md#data-availability-and-access)):**
- `data/cohort1_tororo_all_274_raw_ensembl.csv` — raw RNA-seq counts,
  RESERVE-U-2-TOR (Tororo discovery cohort; controlled access via dbGaP, phs003914.v1.p1)

**Annotation files (included in repository):**
- `annotations/reactomeGS.rda`
- `annotations/MSigDB_C2_human.rda`
- `annotations/MSigDB_C5_human.rda`

**Outputs written to `results/tororo/GSVA/`:**
- `cohort_tororo_GSVA_reactome.tsv`
- `cohort_tororo_GSVA_msigdb_c2.tsv`
- `cohort_tororo_GSVA_msigdb_c5.tsv`
- `sessionInfo.txt`

## Step 2 — Multi-omic factor analysis

### `MOFA_analysis.R`
Preprocesses multi-omic input data and trains MOFA+ models on the RESERVE-U-2-TOR (Tororo discovery cohort). 
Runs multiple independent training iterations and selects the best model by ELBO.

**Source data required (obtain per [Data Availability and Access](../README.md#data-availability-and-access)):**
- `data/cohort1_tororo_all_298_clinical.csv` — clinical metadata, RESERVE-U-2-TOR
  (contact mjc2244@columbia.edu)
- `data/cohort1_tororo_all_274_raw_ensembl.csv` — raw RNA-seq counts, RESERVE-U-2-TOR
  (controlled access via NCBI dbGaP, phs003914.v1.p1)
- `data/cohort1_tororo_all_274_immune_cells_relative_cibersortx.csv` —
  CIBERSORTx immune cell fractions, RESERVE-U-2-TOR
  (contact mjc2244@columbia.edu)
- `data/cohort1_tororo_all_287_olink_proteomics.csv` — Olink proteomics, RESERVE-U-2-TOR
  (available on Dryad, doi:10.5061/dryad.b2rbnzsq2)

**Intermediate files required (generated by `ssEnrichmentScores.R`):**
- `results/tororo/GSVA/cohort_tororo_GSVA_reactome.tsv`
- `results/tororo/GSVA/cohort_tororo_GSVA_msigdb_c2.tsv`
- `results/tororo/GSVA/cohort_tororo_GSVA_msigdb_c5.tsv`

**Outputs written to `results/MOFA_runs/<run_folder>/`:**
- `model.hdf5` — trained MOFA+ model (one per run)
- `factor_scores.csv` — sample-level factor scores (best run)
- `elbo_scores_<n>_trials.csv` — ELBO comparison across all runs
- `sessionInfo.txt`

## Step 3 — LASSO cross-cohort validation

### `validation_lasso.R`
Trains LASSO models predicting MOFA Factor 1 scores using single omics views
(RNA-seq or Olink proteomics, separately) in the RESERVE-U-2-TOR and
evaluates transferability to the RESERVE-U-1-EBB (Entebbe validation cohort).

**Source data required (obtain per [Data Availability and Access](../README.md#data-availability-and-access)):**
- `data/cohort1_tororo_all_274_raw_ensembl_only.csv` — RNA-seq counts, RESERVE-U-2-TOR
  (controlled access via NCBI dbGaP, phs003914.v1.p1)
- `data/cohort2_entebbe_all_128_raw_ensembl.csv` — RNA-seq counts, RESERVE-U-1-EBB
  (openly available via NCBI SRA, BioProject PRJNA794277)
- `data/cohort1_tororo_all_287_olink_proteomics.csv` — Olink proteomics, RESERVE-U-2-TOR
  (available on Dryad)
- `data/cohort2_entebbe_all_278_olink_proteomics.csv` — Olink proteomics, RESERVE-U-1-EBB
  (available on Dryad)

**Intermediate files required (generated by `MOFA_analysis.R`):**
- `results/MOFA_runs/<run_folder>/model.hdf5` — MOFA+ model on RESERVE-U-2-TOR
- `results/MOFA_runs/Entebbe_woGSVA_denovo/model.hdf5` — de novo model on RESERVE-U-1-EBB

**Outputs written to `results/validation_runs/`:**
- `intVal_Factor<n>_<view>_woGSEA.rds` — internal CV results (per factor/view)
- `extVal_Factor<n>_<view>_woGSEA.rds` — external validation results
- `lasso_cv_seeds_<factor>.log`
- `external_lasso_seeds_<factor>.log`

### `validation_lasso_review.R`
Post-processes LASSO results from `validation_lasso.R`. Extracts non-zero
features at lambda.min and exports combined factor score matrices.

**Intermediate files required (generated by `validation_lasso.R`):**
- `results/validation_runs/extVal_Factor<n>_<view>_woGSEA.rds`
- `results/MOFA_runs/Entebbe_woGSVA_denovo/model.hdf5`

**Outputs written to `results/validation_runs/`:**
- `non0_rna_Factor1_lambdaMin_woGSEA.csv`
- `non0_rna_Factor2_lambdaMin_woGSEA.csv`
- `non0_protein_Factor1_lambdaMin_woGSEA.csv`
- `non0_protein_Factor2_lambdaMin_woGSEA.csv`

## Step 4 — RFE signature selection and external validation

### `validation_rfe_feature_selection.R`
Selects compact gene and protein signatures via Recursive Feature Elimination
(RFE) with linear models, evaluates each signature size, and validates the
selected signatures in the Entebbe cohort.

**Source data required (obtain per [Data Availability and Access](../README.md#data-availability-and-access)):**
- `data/cohort1_tororo_all_274_raw_ensembl.csv` — RNA-seq counts, RESERVE-U-2-TOR
  (controlled access via NCBI dbGaP, phs003914.v1.p1)
- `data/cohort2_entebbe_all_128_raw_ensembl.csv` — RNA-seq counts, RESERVE-U-1-EBB
  (openly available via NCBI SRA, BioProject PRJNA794277)
- `data/cohort1_tororo_all_287_olink_proteomics.csv` — Olink proteomics, RESERVE-U-2-TOR
  (available on Dryad)
- `data/cohort2_entebbe_all_278_olink_proteomics.csv` — Olink proteomics, RESERVE-U-1-EBB
  (available on Dryad)

**Intermediate files required:**
- `results/MOFA_runs/Tororo_woGSVA_selected/model.hdf5`
  (generated by `MOFA_analysis.R`)
- `results/MOFA_runs/Entebbe_woGSVA_denovo/model.hdf5`
  (generated by `MOFA_analysis.R`)
- `results/validation_runs/non0_rna_Factor1_lambdaMin_woGSEA.csv`
  (generated by `validation_lasso_review.R`)
- `results/validation_runs/non0_protein_Factor1_lambdaMin_woGSEA.csv`
  (generated by `validation_lasso_review.R`)

**Outputs written to `results/validation_RFE/gene/` and `.../protein/`:**
- `rfe_<type>_rankings_factor1.csv`
- `performance_by_size_factor1.csv`
- `final_signature_factor1.csv`
- `final_model_coefficients_factor1.csv`
- `external_predictions_factor1.csv`
- `validation_summary_factor1.csv`
- `complete_results_factor1.rds`
- `rmse_by_size_factor1.png`
- `external_validation_scatter_factor1.png`

Additional outputs (gene pipeline only):
- `results/validation_RFE/gene/tororo_preprocessed_rna.rds`
- `results/validation_RFE/gene/tororo_factor1_scores.rds`

### `validation_rfe_external.R`
Provides the validation framework functions used for applying the MoSS
signature to external cohorts. **Not run directly**; sourced by scripts in
`MoSS-src/`.

## Step 5 — Single-cell RNA-seq analysis

### `scRNAseq.R`
End-to-end scRNA-seq pipeline: per-sample QC, doublet removal (scDblFinder),
Seurat integration, clustering, cell type annotation, and participant-level
MoSS score projection.

**Source data required (obtain per [Data Availability and Access](../README.md#data-availability-and-access)):**
- Raw RESERVE-U-2-TOR scRNA-seq data (controlled access via NCBI dbGaP, phs004100.v1.p1).
  Set the `data_dir` configuration variable at the top of the script to the
  directory containing these files.

**Intermediate files required:**
- A MOFA Factor 1 score CSV (e.g., `results/MOFA_runs/<run_folder>/factor_scores.csv`,
  generated by `MOFA_analysis.R`) is loaded interactively via `file.choose()`
  for MoSS projection onto the single-cell UMAP.

**Outputs written to the user-specified `out_dir`:**
- `seurat_objects.rda` — pre-QC, post-QC, and QC summary objects
- QC diagnostic PNGs (violin and scatter plots per sample)
- Integrated Seurat object with UMAP, cluster markers, and MoSS projection plots
