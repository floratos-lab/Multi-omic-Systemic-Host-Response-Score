# MoSS: A Multi-omic Host-Response Axis for Stratification of Severe Infection and Critical Illness

This repository contains the analysis code for derivation, validation, and external
projection of the Multi-omic Systemic Host-Response Score (MoSS).
MoSS is a multi-omic latent factor derived from integrated RNA-seq, proteomics,
and immune cell composition data that captures a coordinated host-response axis
of sepsis and severe infection.

## Analytical Workflow

1. **Preprocess input data**
   - Prepare whole-blood RNA-seq, Olink serum proteomics, and CIBERSORTx-derived immune-cell fraction data.
   - Standardize features and align samples across molecular views.

2. **Generate single-sample pathway enrichment scores**
   - Compute GSVA/ssGSEA pathway enrichment scores from whole-blood RNA-seq data.
   - Use these scores as supplementary views in pathway-augmented MOFA+ analyses to improve biological interpretation.

3. **Derive MoSS using multi-omic integration in RESERVE-U-2-TOR (Tororo discovery cohort)**
   - Apply MOFA+ to transcriptomic, proteomic, and inferred immune-cell composition views in RESERVE-U-2-TOR.
   - Identify Factor 1 as the dominant cross-modal host-response axis, termed MoSS.
   - Run additional MOFA+ models to assess robustness and improve biological interpretation:
     - a pathway-augmented model incorporating GSVA/ssGSEA enrichment scores as additional views;
     - a model restricted to directly quantified views only (gene and protein expression).

4. **Validate MoSS in RESERVE-U-1-EBB (Entebbe validation cohort)**
   - Perform de novo MOFA+ modeling in RESERVE-U-1-EBB.
   - Compare de novo MoSS values with projected values from single-view LASSO models trained in RESERVE-U-2-TOR.

5. **Build compact projection models**
   - Use LASSO feature pre-selection followed by recursive feature elimination (RFE) in RESERVE-U-2-TOR.
   - Select compact transcriptomic signatures, resulting in 14-gene RNA-seq model used for external projection.
   - Evaluate model performance in RESERVE-U-1-EBB.

6. **Perform single-cell analyses in RESERVE-U-2-TOR**
   - Process and analyze scRNA-seq data from peripheral blood cells.
   - Annotate major immune populations and resolve monocyte and low-density neutrophil states.
   - Map participant-level MoSS values onto the single-cell landscape to localize MoSS-associated biology.
  
7. **Project MoSS into external bulk RNA-seq cohorts**
   - Apply the 14-gene model to independent whole-blood RNA-seq datasets.
   - Generate per-sample projected MoSS values and gene-mapping success rates.

8. **Further Analyses**
   - Estimates of single-cell MoSS balance in external cohorts were performed separately.

## Data Availability and Access

Source data files are not included in this repository. See the descriptions
below for how to obtain each dataset referenced by the analysis scripts.

### Whole-blood RNA-seq

Raw whole-blood RNA-seq count matrices for the RESERVE-U-2-TOR (discovery) cohort
are available under controlled access via NCBI dbGaP:
[phs003914.v1.p1](https://dbgap.ncbi.nlm.nih.gov/beta/study/phs003914.v1.p1/#study).
To request access, create an eRA Commons account and submit a Data Access
Request through the dbGaP Authorized Access portal.

Raw whole-blood RNA-seq count matrices for the RESERVE-U-1-EBB (validation) cohort
are available via NCBI SRA:
[BioProject PRJNA794277](https://www.ncbi.nlm.nih.gov/bioproject/PRJNA794277).

### Serum proteomics

Olink serum proteomics data for RESERVE-U-2-TOR and RESERVE-U-1-EBB are available on Dryad:
[doi:10.5061/dryad.b2rbnzsq2](https://datadryad.org/dataset/doi:10.5061/dryad.b2rbnzsq2).

### Single-cell RNA-seq

Raw scRNA-seq data for the analyzed RESERVE-U-2-TOR subset are available under
controlled access via NCBI dbGaP:
[phs004100.v1.p1](https://dbgap.ncbi.nlm.nih.gov/beta/study/phs004100.v1.p1/#study).

### CIBERSORTx-derived immune cell fraction estimates 

Estimates of immune cell fractions can be generated from whole-blood RNA-seq data
using CIBERSORTx (LM22 signature matrix) as per methods described in the above 
manuscript. 

### Clinical metadata

De-identified clinical metadata may be made available to researchers affiliated with an appropriate
institution following mutual signing of a data access agreement and obtainment
of necessary ethics approvals. Requests can be sent to mjc2244@columbia.edu.

## Usage

### Full pipeline (src/)

Scripts in `src/` must be run in the order listed in
[src/README.md](src/README.md). All file paths are resolved using `here()`
relative to the repository root — open the `.Rproj` file in RStudio before
running any script.

### External MoSS scoring (MoSS-src/)

Users wishing to predict MoSS values in an external cohort using whole-blood
RNA-seq data should start with the `MoSS-src/` folder. See
[MoSS-src/README.md](MoSS-src/README.md) for configuration instructions.

## Requirements

All scripts are written in R. Required packages:

```r
# Multi-omic analysis
MOFA2, DESeq2, GSVA

# Signature selection
glmnet, caret

# Single-cell analysis
Seurat, scDblFinder

# Gene ID mapping
org.Hs.eg.db, AnnotationDbi

# General
tidyverse, here, GEOquery, readxl
```

Install via Bioconductor and CRAN:

```r
if (!require("BiocManager")) install.packages("BiocManager")
BiocManager::install(c("MOFA2", "DESeq2", "GSVA", "org.Hs.eg.db",
                       "AnnotationDbi", "scDblFinder"))
install.packages(c("glmnet", "caret", "tidyverse", "here", "GEOquery", "readxl"))
# Seurat: see https://satijalab.org/seurat/articles/install_v5
```
